Supervictor ESP-IDF Firmware

ESP32-C3ESP-IDF v5.4mTLS

C-based firmware for ESP32-C3 that sends mTLS-authenticated uplink messages to the Supervictor cloud API.

Quick Start

cd esp-idf
idf.py set-target esp32c3
idf.py menuconfig          # Set WiFi SSID/Password, Host
idf.py build flash monitor

Application Flow

  1. Initialize NVS, WiFi STA, event handlers
  2. Spawn uplink_task (FreeRTOS)
  3. Wait for WiFi + DHCP IP
  4. Loop every 5s: serialize JSON, open TLS to HOST:443, POST, parse response

Configuration

Set via idf.py menuconfig → Supervictor:

  • SV_HOST — Cloud API hostname
  • SV_WIFI_SSID — WiFi network name
  • SV_WIFI_PASSWORD — WiFi password

Constants (buffer sizes, timeouts) in main/config.h.

Certificates

Embedded at build time via CMake EMBED_TXTFILES. Place certs at:

  • ../certs/AmazonRootCA1.pem (CA)
  • ../cloud/certs/devices/test-device/client.pem (device cert)
  • ../cloud/certs/devices/test-device/client.key (device key)

Generate with: ./cloud/scripts/gen_certs.sh device <name>

Testing

Unit tests use Unity (bundled with ESP-IDF). Pure-function tests run on the Linux host target:

cd esp-idf/host_test
idf.py --preview set-target linux
idf.py build
./build/host_test.elf
