CLI Pipeline

The qs CLI orchestrates build, test, and deploy through three progressive stages.

CommandWhat it does
./qs dev Unit tests, SAM build, start sam local, run local integration tests
./qs dev --serve Same as above but keeps sam local running for manual testing
./qs staging Dev gate → deploy to dev stack → integration tests against live endpoint
./qs prod Dev gate → staging gate → confirm → deploy prod → mTLS verification

All commands accept -v (verbose) and --dry-run flags.

Testing Tiers

Unit

36 tests, no infra required. Validates handler logic, Pydantic models, response schemas.

uv run pytest tests/unit/

Local Integration

HTTP against sam local start-api. Requires Docker. Marker: local

./scripts/run_integration_tests.sh

Remote / mTLS

HTTPS + client certs against deployed prod endpoint. Marker: remote

API_ENDPOINT=https://... pytest -m remote

Environments

Production

Certificate Management

API Gateway validates client certificates against a CA truststore in S3. 

./scripts/gen_certs.sh ca                # Create CA
./scripts/gen_certs.sh device mydevice   # Issue device cert
./scripts/gen_certs.sh person alice      # Issue person cert

aws s3 cp certs/ca/ca.pem s3://supervictor/truststore.pem

Quick Start

Clone and install dependencies

git clone git@github.com:JimothyJohn/supervictor.git
cd supervictor/supervictor-cloud
uv sync

Configure your environment

cp .env.example ../.env.dev
# Edit .env.dev with your WiFi credentials and serial port

Run the dev pipeline

./qs dev

Deploy to production (when ready)

./qs prod

CI/CD

GitHub Actions runs on every push to supervictor-cloud/. Two sequential jobs: Unit Tests (no Docker) then Integration Tests (Docker + SAM CLI). Toolchain: uv with caching via astral-sh/setup-uv.
