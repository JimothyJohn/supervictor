CLI Pipeline

The qs CLI orchestrates build, test, deploy, and device management through progressive stages.

CommandWhat it does
qs dev Unit tests, build endpoint, start local server, run integration tests
qs dev --serve Same as above but keeps the server running for manual testing
qs edge Build + flash ESP32-C3 firmware via espflash 3.3.0
qs staging Dev gate → deploy to dev stack → integration tests against live endpoint
qs prod Dev gate → staging gate → confirm → deploy prod → mTLS verification
qs certs ca Initialize the root CA for mTLS
qs certs device <name> Issue a device client certificate
qs certs server <name> Issue a server/TLS certificate
qs certs verify Verify the mTLS certificate chain
qs certs handshake Simulate mTLS handshake against a running server
qs ping mTLS GET to verify the server is reachable
qs onboard End-to-end device onboarding (certs + flash + verify)

All commands accept -v (verbose) and --dry-run flags.

Testing Tiers

Unit

In-memory SQLite, no infra required. Validates handlers, models, and store logic.

cargo test --features sqlite

Local Integration

HTTP against the running endpoint. Requires Docker + Caddy for mTLS.

docker compose up

Remote / mTLS

HTTPS + client certs against the deployed production endpoint.

qs ping --certs certs/device/

Environments

Dev

Local endpoint via docker compose with Caddy mTLS reverse proxy

Staging

Deployed dev stack on execute-api, no mTLS enforcement

Production

Custom domain supervictor.advin.io with mTLS, Route53, ACM certificate

Certificate Management

The qs certs subcommand manages the full mTLS certificate lifecycle. 

qs certs ca                          # Create root CA
qs certs device esp32                # Issue device cert
qs certs server caddy                # Issue server cert
qs certs verify                      # Verify cert chain
qs certs handshake                   # Simulate mTLS handshake

Quick Start

Clone the repository

git clone git@github.com:JimothyJohn/supervictor.git
cd supervictor

Build the CLI

cd cli && cargo build --release
alias qs=./target/release/qs

Generate certificates and start the local endpoint

qs certs ca && qs certs device esp32 && qs certs server caddy
qs dev --serve

Flash the device

qs edge

Deploy to production (when ready)

qs prod
